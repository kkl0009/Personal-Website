Computer Forensics Investigation

The main report The "kitty" report My class portfolio

About: This was a computer forensics assignment I had as my final project in my computer incident response course. This was an individual assignment where we were each given a Windows 7 virtual machine, and our task was to get access to the machine (by exploiting a vulnerability in Windows 7) and to document any evidence of a fake driver's license operation in a formal forensic report. Additionally, the computer had several instances of "kitty porn", which were pictures of cats meant to represent illegal material. During the investigation, upon finding such material, we were required to receive a warrant from our teacher and write a separate report on the illegal material (also linked above). These 2 reports detail all material found on the system and organizes the findings in chronological order to create an attack narrative. I highly recommend taking a look at least at the main report above if you have the time.

In addition to my forensic reports, I have included a copy of my portfolio for this incident response course. The most significant portion of this portfolio is a list of security tools we used during the class. This portfolio also include documentation of my National Cyber League placement of 180th worldwide.

Computer Network Penetration Test

Our final report

About: This was my final project for my penetration testing course (CYBE 467). I was on a team of 6 people, and our goal was to perform a full penetration test on a network of computers which were set up at West Virginia University. This network was made up of multiple subnetworks, each containing multiple computers. The penetration test involved scanning systems with Nmap, password cracking with John the Ripper and Hashcat, connecting to machines using Secure Shell and Remote Desktop Protocol, examining vulnerable web applications, searching file systems, looking through MySQL databases, and more. The machines were a mix of Linux and Windows machines, and the test was conducted using a Kali Linux virtual machine. I strongly encourage you to take a look at our final report (linked above) for screenshots and more details on what we found.

Buffer Overflow Lab

The lab report

About: This was a lab report I completed in one of my cybersecrity courses. The object of the lab was to use a buffer overflow vulnerability in a C program to gain a root shell on a Linux virtual machine. The was accomplished by using GNU Debugger to manipulate specific locations in memory and insert a shellcode (written in Assembly). As can be seen in the lab report, I was able to get a root shell, although it was not easy given how specific the shellcode needed to be.

Morgantown Local Restaurants Web Application

Github repository Our final presentation

About: This was one of my first experiences with web development. It is a website which was hosted on Amazon Web Services Lightsail. The purpose of the website was to allow students and other people who are new to Morgantown, WV find local restaurants to eat at. The website includes about 30 restaurants, each stored in a JSON file which can be easily updated. The site was created primarily using PHP and HTML but there was some JavaScript used in it as well. Users could create and access individual accounts via a SQL database with PHPMyAdmin. They could use these accounts to write reviews of any restaurants on the website. The website was created on a team of 5 people.
